# 위탁판매 통합 자동화 시스템 v2

오너클랜 상품 소싱부터 쿠팡 마진검증, 상세페이지 자동생성, 상품등록, 그리고 판매 성과 기반 최적화(하차/가격조정)까지 전 과정을 5개 모듈로 통합한 자동화 파이프라인 시스템입니다.

## 📋 핵심 기능 (5개 모듈)

1. **A. 소싱 및 마진 검증**: 오너클랜 API에서 상품을 수집하고, 수수료/VAT/반품비 등을 고려한 정교한 마진율을 계산하여 필터링합니다.
2. **B. 상세페이지 자동 생성**: Jinja2 HTML 템플릿과 LLM(Claude)을 활용하여 쿠팡 규격에 맞는 고퀄리티 상세페이지와 필수 고시정보를 자동 생성합니다.
3. **C. 상품 등록 (반자동)**: 생성된 상세페이지를 관리자가 대시보드에서 한 번 확인(승인)한 후 쿠팡 API를 통해 판매 등록합니다.
4. **D. 판매 성과 모니터링**: 쿠팡 발주서/정산 데이터를 수집하여 실제 마진율과 상품별 판매량을 집계합니다.
5. **E. 최적화 루프**: 판매율 저조(예: 14일 0건) 상품은 자동 하차 후보로 분류하고, 도매가 인상 시 가격 조정 알림을 보냅니다.

## 🛠️ 기술 스택

- **Backend**: Python 3.10+, FastAPI, SQLAlchemy (Async), SQLite
- **Frontend**: React 18, Vite, Recharts, CSS Glassmorphism UI
- **Scheduler**: APScheduler (내장)
- **API 연동**: Coupang Open API (HMAC), Ownerclan API, Anthropic (Claude) API

## 🚀 로컬 실행 방법

이 시스템은 현재 **Mock 모드**로 동작하도록 기본 설정되어 있습니다. API 키 없이도 전체 파이프라인을 체험해 볼 수 있습니다.

### 1. 백엔드 실행

```powershell
cd backend

# 가상환경 생성 및 활성화 (선택)
python -m venv venv
.\venv\Scripts\activate

# 패키지 설치
pip install -r requirements.txt

# 환경변수 파일 복사 (기본 설정으로 충분)
copy .env.example .env

# 서버 실행 (기본 포트: 8000)
python run.py
```

- Swagger API 문서: `http://localhost:8000/docs`

### 2. 프론트엔드 대시보드 실행 (새 터미널 열기)

```powershell
cd frontend

# 패키지 설치
npm install

# 개발 서버 실행 (기본 포트: 5173)
npm run dev
```

- 대시보드 접속: `http://localhost:5173`

## 💡 Mock 모드 사용 가이드 (시나리오)

실제 API 연동 없이 프론트엔드 UI를 통해 파이프라인을 테스트하는 순서입니다.

1. **소싱·마진검증 탭**: `[소싱 스캔 실행]` 버튼을 누릅니다. Mock 상품 8개가 불러와지며 마진율 계산 후 합격/불합격이 나뉩니다.
2. **상품관리·등록 탭**:
   - 상단 `[소싱완료 상품 상세페이지 일괄 생성]` 버튼을 눌러 모듈 B를 실행합니다.
   - 상태가 "페이지생성완료"로 바뀐 상품 리스트에서 `[쿠팡 등록 승인]`을 누릅니다.
   - 상태가 "등록완료"로 바뀌면 `[판매 시작]`을 누릅니다.
   - "판매중"인 상품은 `[미리보기]` 버튼으로 자동 생성된 HTML 상세페이지를 확인할 수 있습니다.
3. **판매 모니터링 탭**: 상단의 `[주문 데이터 수집]` 버튼을 눌러 랜덤 생성된 Mock 주문 데이터를 가져옵니다. 상품별 매출과 순이익이 계산됩니다.
4. **상품관리·등록 탭 (최적화)**: 상단의 `[최적화 루프 실행]` 버튼을 누릅니다. 판매가 없거나 마진이 부족한 상품이 "하차후보" 및 "가격조정필요" 상태로 자동 태깅됩니다.
5. **대시보드 탭**: 전체 KPI 지표와 매출 트렌드 차트를 확인합니다.
